

# Fix SEO Issues: Canonicals, H1 Tags, Internal Linking, and Sitemap Sync

## Issues Identified

1. **No canonical tags on any page** — `SEOHead` supports `canonical` prop but no page passes it. Search engines see sitemap URLs without matching canonical declarations.

2. **Sitemap contains pages not generated by the app** — The static `sitemap.xml` may include combo slugs that don't match the dynamically generated `comboPages`. Need to sync sitemap with actual `getAllSlugs()`.

3. **Orphan pages** — Many combo pages (e.g., `/buy-usdt-india-upi`) have no internal links pointing to them except the sitemap. Footer only links to 8 combo pages.

4. **HTTP compression / caching** — These are server-level concerns. Cannot be fixed from client code on Lovable hosting. The `_headers` file only works on Netlify/Cloudflare.

## Plan

### Step 1: Add canonical URLs to all pages

Update `SEOHead` usage across all pages to pass `canonical`:

- **`SEOLanding.tsx`** — Already has the URL string; pass `canonical={https://buysusdtp2p.com/${page.slug}}`
- **`Index.tsx`** — `canonical="https://buysusdtp2p.com/"`
- **`Marketplace.tsx`** — `canonical="https://buysusdtp2p.com/marketplace"`
- **`Blog.tsx`** — `canonical="https://buysusdtp2p.com/blog"`
- **`BlogPost.tsx`** — `canonical={https://buysusdtp2p.com/blog/${slug}}`
- **`Fees.tsx`** — `canonical="https://buysusdtp2p.com/fees"`
- **`HowItWorks.tsx`** — `canonical="https://buysusdtp2p.com/how-it-works"`
- **`Auth.tsx`** — `canonical="https://buysusdtp2p.com/auth"`
- **`Verify.tsx`** — `canonical="https://buysusdtp2p.com/verify"`
- **`Dashboard.tsx`**, **`TradePage.tsx`**, **`AdminRisk.tsx`** — No canonical needed (authenticated/dynamic pages)

### Step 2: Auto-generate sitemap.xml from SEO page data

Replace the static `public/sitemap.xml` with a build-time script or a dynamically generated file that uses `getAllSlugs()` plus the static routes. This ensures the sitemap always matches actual app routes.

Create a **Vite plugin** or **build script** that writes `sitemap.xml` into the `dist/` folder at build time, pulling slugs from `seo-pages.ts`.

Alternatively, regenerate the static file now to match all current slugs (combo pages included).

### Step 3: Improve internal linking to reduce orphan pages

- **Footer**: Expand the "Popular Searches" section to include more combo links, or add a dedicated "Browse by Country + Payment" section
- **SEO Landing pages**: Each page already has `relatedLinks` — verify these render as actual `<Link>` elements (they do)
- **Index.tsx**: Add a section linking to top country/payment landing pages

### Step 4: Verify H1 tags on all pages

Quick audit shows all pages already use `<h1>` tags. No changes needed here — the SEO tool may be flagging specific pages; all main pages have proper H1s.
